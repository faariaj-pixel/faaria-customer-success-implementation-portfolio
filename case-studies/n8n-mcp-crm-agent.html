Case study

n8n MCP CRM Agent

A two-workflow AI CRM system that lets users create, read, update, and delete Google Sheets records through natural language.

Contact FaariaBack to portfolio

Challenge

AI agents that modify business records need clear tool boundaries and safeguards to prevent accidental updates, deletes, or duplicate actions.

Approach

  • Separated the user-facing AI Agent from the MCP Server workflow.
  • Used Google Sheets CRUD tools behind an MCP Client.
  • Required Get Rows before update/delete actions.
  • Used email as a unique identifier for safer record selection.

Outputs

  • Agent-side n8n workflow
  • MCP Server workflow
  • Google Sheets CRUD tools
  • Safety prompt rules for update/delete

Relevance to Customer Success & Implementation

Relevant to customer success and implementation because it demonstrates safeguards, tool-use clarity, and controlled actions for workflows that touch customer or CRM records.
